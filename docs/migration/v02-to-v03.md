# Migrating from v0.2.x to v0.3.0

v0.3.0 introduces a new top-level API (`lmdiff.compare`, `lmdiff.family`),
a frozen-and-validated `Config` dataclass with typed sub-specs, and a
backend-agnostic `Engine` Protocol. The pre-v0.3.0 surfaces continue to
work for one release with `DeprecationWarning`; both are removed in
**v0.4.0**.

## What changed

- **`lmdiff.Config`** is now the v0.3.0 dataclass at `lmdiff._config.Config`.
  It is `frozen=True`, hashable, validated at construction, and uses typed
  sub-specs (`DecodeSpec`, `AdapterSpec`, `QuantSpec`, `PruneSpec`,
  `KVCacheSpec`, `SteeringSpec`, `ICLExample`, `Message`).
- **`lmdiff.compare(base, variant, ...)`** and
  **`lmdiff.family(base, {name: variant, ...}, ...)`** are the new
  one-call entry points. They accept either model-id strings or `Config`
  instances, build engines internally, and clean up after themselves.
- **`Engine`** is a `runtime_checkable` Protocol. The canonical
  implementation is `HFEngine`; bring your own backend via `MinimalEngine`.
- **The pre-v0.3.0 `lmdiff.config.Config`** is the old dataclass with
  `decode: dict`, `dtype: str | None`, `ttt`, etc. It still works but
  emits `DeprecationWarning` on construction.
- **The pre-v0.3.0 `lmdiff.ModelDiff`** is the old wrapper. It still
  works but emits `DeprecationWarning` on construction;
  `ModelDiff.run(level="representation")` raises `NotImplementedError`
  pointing at v0.7.0 / Phase 5.

## Quick mapping table

| v0.2.x                                                              | v0.3.0                                                                 |
|---------------------------------------------------------------------|------------------------------------------------------------------------|
| `from lmdiff.config import Config`                                  | `from lmdiff import Config`                                            |
| `Config(model=..., decode={"strategy": "sample", "temperature": 0.7})` | `Config(model=..., decode=DecodeSpec(strategy="sample", temperature=0.7))` |
| `Config(model=..., dtype="bfloat16")`                               | (handled by Engine, not Config; see `HFEngine(dtype="bf16")`)          |
| `ModelDiff(a, b, prompts, n_samples=N).run()`                       | `lmdiff.compare(a, b, probes=prompts, n_probes=N)`                     |
| `ModelDiff(a, b, prompts).run(level="output")`                      | `lmdiff.compare(a, b, probes=prompts)` (default = output)              |
| `ModelDiff(a, b, prompts).run(level="representation")`              | `NotImplementedError` until v0.7.0 (Phase 5: representation metrics)   |
| `ChangeGeometry(base, {n: v}, prompts).analyze()`                   | `lmdiff.family(base, {n: v}, probes=prompts)`                          |

## Deprecation timeline

- **v0.3.0** (current): both surfaces active. `lmdiff.config.Config` and
  `lmdiff.ModelDiff` emit `DeprecationWarning` on construction. Old
  callers continue to work unchanged otherwise.
- **v0.4.0**: removal. `lmdiff.config` module deleted; `lmdiff.ModelDiff`
  removed from the top-level namespace.

If you need more time, pin `lmdiff-kit<0.4` in your `pyproject.toml`.

## Common gotchas

1. **`Config.decode` is no longer a dict.** Pass a `DecodeSpec`. Common
   recipe:
   ```python
   from lmdiff import DecodeSpec
   DecodeSpec(strategy="sample", temperature=0.7, top_p=0.95, max_new_tokens=64)
   ```
2. **`Config.dtype` is gone.** Precision is an Engine concern, not a
   Config concern. Pass `dtype="bf16"` to `HFEngine(...)` if needed; the
   default is `"bf16"`.
3. **`from lmdiff import Config` now resolves to the new class** even if
   you don't migrate everything else. `from lmdiff.config import Config`
   stays bound to the old class for the deprecation window.
4. **`compare()` and `family()` close engines they create themselves**
   in a `finally` block, so accidentally erroring mid-run no longer leaks
   GPU memory. If you pass `engine=...` as a template, you keep
   ownership.

## Where to look

- New `Config` and sub-specs: `lmdiff/_config.py`.
- New `Engine` protocol + `HFEngine`: `lmdiff/_engine.py`.
- New `compare()` / `family()`: `lmdiff/_api.py`.
